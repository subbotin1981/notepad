День 1. Первая страница

Зачем нужны теги и как их заводить?

Теги в HTML нужны чтобы управлять содержимым и структурой сайта. Потому что браузер не понимает переносов строк, списков и всего прочего без этих самих тегов

Теги задаются при помощи специальных угловых кавычек

Синтаксис

<название тега>

Все теги HTML делятся на две группы:

  1. Парные теги Называются так потому что работают в паре. Первый тег называется - открывающий. Второй - закрывающий. В нем добавляется слеш.
  2. Непарные теги Называются так потому что не имеют пары. Это одиночные теги, в них нет содержимого и закрывающий тег им не нужен.

Атрибуты тегов

У каждого тега могут быть свои атрибуты которые отвечают за его свойства. Они задаются внутри тега и имеют следующий вид:

Название атрибута - его значение

Пример использования

<hr width="300">

в примере выше мы указали что линия должна быть шириной в 300 пикселей

У одного тега может быть множество атрибутов и все они задаются через пробел.

Также, значения атрибутов могут задаватся как через двойные кавычки, так и через одиночные. Или вообще без кавычек.

Если тег парный - атрибуты задаются только для первого, открывающего тега.

ВАЖНО:

Всегда используйте один тип задания значений для атрибутов. Чтобы не путаться и не делать лишних ошибок. В этом тренинге мы будем задавать значения для атрибутов только при помощи двойных кавычек.

Директива DOCTYPE

Директива DOCTYPE используется для того, что-бы определить тип документа и указать браузеру - как обрабатывать нашу страничку. А точнее - какая версия HTML использовалась при ее создании.

Например, в нашем случае - DOCTYPE указывает браузеру, что страница создается на HTML5 и что ее надо обрабатывать в строгом соответствии этому стандарту.

День 2. Форматирование

Спецсимволы в HTML

Используются для того, чтобы отобразить спецсимволы на сайте которые изначально обрабатываются браузером как код и не отображаются посетителям по умолчанию.

Пример использования:

<!DOCTYPE html>

задаются при помощи знака амперсанд & затем идет название спецсимвола и заканчивается точкой с запятой.

&lt; - в итоге у нас получится вот такая угловая кавычка: <

Так же мы можем вставлять специальные знаки, которые изначально невозможно задать при помощи клавиатуры

&copy; - в итоге у нас получится значок копирайта ©

Тег sub этот текст идет в - <sub> нижнем индексе</sub>

День 3. Списки и ошибки

Нумерованый список

Называется он так потому, что все его элементы нумеруются по порядку. Причем нумероваться они могут различными переменными

Задается через тег <ol> Все элементы списка добавляются при помощи тега <li> .

Пример использования:

<ol>

  1. <li>Элемент нумерованного списка</li>
  2. <li>Элемент нумерованного списка</li>
  3. <li>Элемент нумерованного списка</li>

</ol>

Маркерованный список

Называется он так потому, что все его элементы помечаются маркерами. Причем маркеры могут быть разные

Задается через тег <ul> Все элементы списка добавляются при помощи тега <li> .

Пример использования:

<ul>

  • <li>Элемент маркированного списка</li>
  • <li>Элемент маркированного списка</li>
  • <li>Элемент маркированного списка</li>

</ul>

Важно

Типы маркеров в обоих списках можно изменить с помощью атрибутов. Но в HTML5 это осуждается так как можно сделать тоже самое при помощи CSS3.

Список определений.

Задаются эти списки при помощу тега <tl>. При чем используются не 2 тега, как в первых списках. А сразу три.

Второй тег <dt> - задает термин.

Второй тег <dd> - задает определение для термина.

Пример использования:

<dl>

<dt>Чекушка</dt>
<dd>используется многими продвинытуми тоариСчами для быстрого утоления жажды (спрайт - отдыхает)</dd>
<dt>Флян</dt>
<dd>используется теми же товарищами если чекушка не спасла от жажды</dd>
<dt>Банка</dt>
<dd>используется в кравних случаях при затажных праздниках</dd>

<dl>

Обязательно надо использовать оттступы. Иначе код будет плохо читаем.

Ссылки и картинки

Ссылки на сайте

ссылки на сайте задаются при помощи тега А, который является парным. Текст между ссылками называется анкором. Сама ссылка задается через атрибут href

Пример использования:

<a href="ссылка"> анкор ссылки </a>

Ссылка на сайт

Ссылки делатся на три вида

1. Внешние ссылки
Ведут на другой сайт
2. Внутренние ссылки
Ведут на другую страницу того же сайта
3. Якоря
Ведут на другой участок той-же страницы

Переход на главную страницу дневника.

Якоря

Якорь задается при помощи атрибута id.Сначала на странице в нужном месте создаем якорь а потом создаем обычгую ссылку, только она будет вести на якорь.

Пример использования:

<a id="якорь"> анкор ссылки </a>

<a href="#якорь"> анкор ссылки </a>

Перейти на верх страницы

Атрибуты для ссылок

href - указывает путь ссылки

rel="nofollow" - поисковики не учитывают ТИЦ и PR который может передавать ссылка

title="подсказка" - всплывающая подсказка при наведении курсора

target - указывает как должна открываться ссылка. По умолчанию атрибут _self - ссылка откорывается в том же окне. Атрибут _blank - откроется в новом окне.

Картинки на сайте

Задаются при помощи тега IMG, который является непарным. Путь к картинке через специальный атрибут src. Расширения картинок, которые можно использовать, это - jpg, png и gif.

Атрибуты для картинок

alt - дополительный специальный атрибут к картинкам. без него будут работать, но код будет не валидным и это плохо скажется на оптимизации. Используется если картинки отключены сайтом тогда отображается текст из этого атрибута

title="подсказка" - всплывающая подсказка при наведении курсора

height и width - если указываается только один артрибут то второй подстроится автоматически под пропорции картинки. Могут задаваться в % и px.

Важно

Если указывать размеры в % - вычисляются не от размеров картинки а от размеров блока картинки

Пример использования:

<img src="путь к картинке" alt="альтернативный текст">

таже самая картинка только с widht="100":

Таблицы

Таблицы задаются при помощи тега table. В каждой таблице есть строки (тег tr)и в каждой строке содержатся ячейки (тег td).

Атрибуты для ячеек:

colspan - объединение ячеек по горизонтали. Используется только целое число выше 1.

rowspan - объединение ячеек по вертикали. Используется только целое число выше 1.

Дополнительные теги:

caption - этот тег отвечает за заголовок таблицы. Задается только внутри таблицы.

th - отвечает за создание загаловочной ячейки. Текст по умолчанию делается жирным и размещается по центру

col - используется что бы указать доп параметры для колонок таблицы. Этот тег не парный! Испльзуется атрибут width и span.
