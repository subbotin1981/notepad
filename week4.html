День 2. Музыкальный плеер на сайте.

Аудиоплеер

Тег<audio> позволяет проигрывать музыкальные композиции на сайте. Путь к странице можно задать двумя способами. Через тег <source>, либо через атрибут src.

<audio src='путь к аудио'></audio>
<audio><source src='путь к аудио'></audio>

Атрибуты музыкального проигрывателя:

controls - указывает что необходимо отображать элементы контроля над плеером.

autoplay - задает автоматическое воспроизведение файла

loop - зацикливает воспроизведение файла

preload - указывает браузеру как нужно подгружать мультимедиа на странице

  • none - не загружать файл
  • auto - стоит по умолчанию, файлы загружаются вместе со страницей
  • metadata -браузер подгружает только информацию о файле мультимедиа

День 3. Видеопроигрыватель на сайте

Видеопроигрыватель:

Тег<video> позволяет проигрывать видео композиции на сайте. Путь к странице можно задать двумя способами. Через тег <source>, либо через атрибут src.

<video src='путь к видео'></video>
<video><source src='путь к видео'></video>

Пример 1:

<video src="media/clip.mp4" controls preload="metadata" height="315" width="560"'></video>

Пример 2:

<video controls="controls" poster="media/poster.jpg" height="315" width="560" ></video>

Пример 3:

<video controls preload="metadata" poster="media/poster.jpg">
   <source src="media/clip.mp4" media="screen" type="video/ogg">
   <source src="media/clip.ogg" media="screen">
</video>

Атрибуты, которые мы рассмотрели к аудиопроигрователю, подходят и к видеопроигрывателю. Но у него есть своих дополнительных атрибутов.

Дополнительные атрибуты видеопроигрывателя:

width - указывает ширину для видео в пикселях или процентах.

height - указывает высоту для видео в пикселях или процентах.

poster - вставляет постер (картинку-заставку) в видео, которое отображается до воспроизведения.

Помните, что пропорции постера и видео должны соответствовать иначе постер будет отображаться не пропорционально и не красиво, будет искажаться

Тег<source> - указывает путь к файлу который необходимо проиграть. Ниже рассмотрим атрибуты этого тега.

  • type - указывает тип воспроизводимого файла.
  • src - задает путь к файлу
  • media - для какого устройства должен производиться этот файл.

Значения атрибута media:

  • all - для всех устройств.
  • handheld - для смартфонов, телефонов, планшетов и тд
  • screen - для экранов монитора
  • speech - для речевых синтезаторов
  • projection - для проекторов
  • tv - для телеков
  • tty - телетайпы
  • print - для печати

Также воспроизведение видео задается через тег <iframe > :

Пример использования:

<iframe>< src='путь к видео' width="560" height="315" frameborder="0" allowfullscreen ></iframe>

День 4. Формы

Формы задаются при помощи тега <form>, который является парным. Содержимое формы на 99% состоит из тегов <input> различного типа. Этот тег является одиночным.

<form action="#" accept-charset="utf-8" method="post" name="test">
      <input name="name">
      <input name="mail">
      <input name="text">
</form>

Атрибуты форм:

  • action - указывает на адрес программы или файла обработчика.
  • accept - устанавливает фильтр на типы файлов, которые вы можете отправить через поле загрузки файлов.
  • accept-charset - устанавливает кодировку для отправленных данных.
  • autocomplete - включает/выключает (on/off)автозаполнение форм (по умолчанию стоит - on).
  • accesskey - Переход к элементу с помощью комбинации клавиш.
  • align - Определяет выравнивание изображения.
  • alt - Альтернативный текст для кнопки с изображением.
  • autocomplete - Включает или отключает автозаполнение.
  • autofocus - Устанавливает фокус в поле формы.
  • border - Толщина рамки вокруг изображения.
  • checked - Предварительно активированный переключатель или флажок.
  • disabled - Блокирует доступ и изменение элемента.
  • enctype - указывает способ кодировки отправленных данных.
  • form - Связывает поле с формой по её идентификатору.
  • formaction - Определяет адрес обработчика формы.
  • formenctype - Устанавливает способ кодирования данных формы при их отправке на сервер.
  • formmethod - Сообщает браузеру каким методом следует передавать данные формы на сервер.
  • formnovalidate - Отменяет встроенную проверку данных на корректность.
  • formtarget - Определяет окно или фрейм в которое будет загружаться результат, возвращаемый обработчиком формы.
  • list - Указывает на список вариантов, которые можно выбирать при вводе текста.
  • max - Верхнее значение для ввода числа или даты.
  • maxlength - Максимальное количество символов разрешенных в тексте.
  • method - имеет два значения (get или post). Указывает как передавать данные.
  • min - Нижнее значение для ввода числа или даты.
  • multiple - Позволяет загрузить несколько файлов одновременно.
  • name - Имя поля, предназначено для того, чтобы обработчик формы мог его идентифицировать. Задает уникальное имя для формы.
  • novalidate - отменяет проверку на правильность заполнения полей форм.
  • pattern - Устанавливает шаблон ввода.
  • placeholder - выводит текст внутри поля формы, который исчезает при получении фокуса.
  • readonly - Устанавливает, что поле не может изменяться пользователем.
  • required - Обязательное для заполнения поле.
  • size - Ширина текстового поля.
  • src - Адрес графического файла для поля с изображением.
  • step - Шаг приращения для числовых полей.
  • target - указывает каким образом будет отображен результат выполнения формы.
  • tabindex - Определяет порядок перехода между элементами с помощью клавиши Tab.
  • type - Сообщает браузеру, к какому типу относится элемент формы.
  • value - Значение элемента.
Помните что имя формы и имена ее элементов должны быть уникальными. Имена задаются произвольно, они могут состоять только из латинских букв, цифр и дефисов.

Почти все элементы формы задаются через тег <input>, и без этого тега форма работать не сможет.

У тега input обязательно должны присутствовать 2 атрибута: name и type.

День 5. Тег input

B HTML5 появилось множество новых полей. Некоторые новые поля еще не корректно работают в браузерах.

Новые значения атрибута TYPE в HTML5

  • color - поле для ввода цвета.
  • email - указывает что в поле должен быть введен адрес эл.почты.
  • number - числовое поле.
  • date - указывает что в этом поле содержиться дата.
  • month - указывает месяц
  • week - указывает неделю
  • datetime - указывает дату и время
  • datetime-local - указывает местное время и дату
  • time - указывает время.
  • url - для ввода адрестной строки
  • tel - поле для телефона.
  • search - поле для поиска информации.
  • range - задает ползунок.

Пример формы:

Форма для заказа киллера

Укажите имя:
Укажите почту:
Выберите дату:
Сколько лет?:

Ваш контакт:
Ваш телефон:
Где искать?:
Выберите цвет глаз:

Будьте внимательны, тк не все типы полей поддерживаются современными браузерами. Обычно если поле не поддерживается оно обраватывается браузером как обычное текстовое поле.

Поля email,number, range, search, tel, url уже поддерживэтиаются всеми современными браузерами и уже можно использовать эти типы не беспокоясь за работоспособность.
